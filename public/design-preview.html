iOS PWA — what's specific

  • Apple ignores manifest.json icons. They read <link rel="apple-touch-icon" href="/apple-touch-icon.png"> — 180×180 PNG, no rounded corners (iOS rounds them).
  • Status bar: add <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent"> so our dark UI bleeds under the iOS notch.
  • Theme color: <meta name="theme-color" content="#13172a"> matches our `--ink-void`.
  • App title on Home Screen: <meta name="apple-mobile-web-app-title" content="War Room">
  • Launch splash screens: need PNGs per device (iPhone 14 Pro, 15, 15 Pro Max, SE, iPad, etc.) — we generate them from one master 2048×2048 PNG via vite-plugin-pwa's splash generator.
  • Web Push: only works on iOS 16.4+ when the user has installed the PWA (Add to Home Screen). Requires user permission. Notification icon shape uses the apple-touch-icon.
  • PWA install: iOS has NO auto-install banner. We need a custom in-app "Install" hint that shows once per session for iOS Safari users.

Android PWA — what's specific

  • Reads manifest.json fully. The 192 + 512 icons (and maskable variants) are required for "Add to Home Screen" prompt.
  • Auto-install banner: Chrome shows the install prompt automatically when criteria met (HTTPS, manifest, service worker, valid icons). We can also trigger it manually with window.deferredPrompt.prompt() bound to a button in Settings.
  • Maskable icons: Android crops adaptive icons into circle/squircle/teardrop depending on launcher. Our maskable PNG must keep the DAD crest inside a 80% center safe zone — anything in the outer 20% can be cropped.
  • Theme color: sets the toolbar color on Android Chrome AND the install-banner accent.
  • Web Push: full Web Push API supported. Background sync for notifications when app closed. Notification badge icon must be monochrome white-on-transparent.
  • Splash screen: auto-generated from manifest theme_color + 512 icon. No separate splash images needed (unlike iOS).
